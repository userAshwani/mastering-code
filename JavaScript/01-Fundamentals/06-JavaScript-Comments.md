**JavaScript comments are notes in code that JavaScript does not execute.**

</div>

---

## Quick Summary

- Comments explain JavaScript code.
- Single-line comments start with `//`.
- Multi-line comments use `/* ... */`.
- Comments can prevent code from running during testing.

---

## Key Points

| Comment Type | Simple Meaning |
| :--- | :--- |
| `//` | Comments one line. |
| `/* ... */` | Comments many lines. |
| End-of-line comment | Adds a note after code. |
| Commented code | Code that is ignored. |

## Remember

- Comments are ignored by JavaScript.
- Use comments to explain code.
- Use comments to make code easier to read.
- Do not keep unnecessary commented code.
